# Recipe Sharing App

A web application that provides users with a one-place solution for browsing and sharing recipes. The platform features both user-facing recipe browsing and admin-controlled recipe management with secure authentication.

## Features

- **Dual Interface**: Separate admin and user interfaces for different user roles
- **Recipe Management**: Add, browse, and search recipes with detailed information
- **Secure Authentication**: Admin login system with session management
- **Responsive Design**: Modern UI with carousel and search functionality
- **Database Integration**: MySQL backend for persistent data storage

## Technology Stack

- **Backend**: Node.js with Express.js framework
- **Database**: MySQL with connection pooling
- **Session Management**: Express-session for user authentication
- **Frontend**: HTML5, CSS3, JavaScript
- **Security**: Environment variable configuration for database credentials

## Database Schema

The application uses a MySQL database with two main tables:

### admin_user Table
- `username` (VARCHAR(50), PRIMARY KEY)
- `u_password` (VARCHAR(50))
- `u_fname` (VARCHAR(50))
- `u_lname` (VARCHAR(50))
- `u_email` (VARCHAR(50))

### recipes Table
- `rcp_id` (INT, AUTO_INCREMENT, PRIMARY KEY)
- `rcp_title` (VARCHAR(100))
- `rcp_cuisine` (VARCHAR(50))
- `rcp_ingredients` (TEXT)
- `rcp_instructions` (TEXT)
- `rcp_cooktime` (VARCHAR(20))
- `rcp_servings` (VARCHAR(20))
- `rcp_author` (VARCHAR(50), FOREIGN KEY references admin_user.username)

## Installation and Setup

### Prerequisites
- Node.js (v14 or higher)
- MySQL Server
- npm package manager

### Setup Instructions

1. **Clone or download the project**

2. **Install dependencies**:
   ```bash
   npm install express mysql body-parser express-session dotenv
   ```

3. **Configure the database**:
   - Create a MySQL database named `recipeLog`
   - Update the `.env` file with your database password:
     ```
     DB_PASSWORD=your_mysql_password
     ```

4. **Create database tables**:
   ```bash
   node create_table.js
   ```

5. **Insert sample admin users** (optional):
   ```bash
   node insert_group_users.js
   ```

6. **Start the application**:
   ```bash
   node server.js
   ```

7. **Access the application**:
   - User interface: http://localhost:3000/homepage.html
   - Admin login: http://localhost:3000/

## Usage

### For Users
- Browse recipes on the homepage
- Search recipes by category or keywords
- View detailed recipe information including ingredients and instructions

### For Admins
- Log in with admin credentials
- Add new recipes with complete details
- Manage recipe content
- Secure logout functionality

## API Endpoints

- `GET /` - Admin login page
- `POST /login` - Admin authentication
- `GET /admin.html` - Admin dashboard (protected)
- `POST /add-recipe` - Add new recipe (protected)
- `GET /recipes` - Get all recipes (JSON)
- `GET /logout` - Logout and destroy session
- `GET /homepage.html` - User recipe browsing interface

- Session-based authentication
